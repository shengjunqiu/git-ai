最快上手：5 步完成安装与使用

按顺序完成以下步骤即可开始使用；遇到问题时再查看后面的详细说明。

1

确认 Git 并注册账号

运行 git --version，确认电脑已经安装 Git；然后使用公司企业邮箱注册账号，并保持浏览器登录状态。

打开注册页面 ↗
2

安装 Git AI

macOS / Linux 在终端执行：

curl -fsSL \
  http://117.147.213.234:38080/worker/releases/latest/download/install.sh \
  | bash

Windows 在 PowerShell 执行：

irm http://117.147.213.234:38080/worker/releases/latest/download/install.ps1 | iex

安装完成后重新打开终端或 PowerShell，并运行 git-ai --version。

3

登录并授权

运行登录命令，在打开的浏览器中确认授权，再检查当前账号：

git-ai login --server http://117.147.213.234:38080
git-ai whoami

git-ai whoami 能显示当前账号即表示授权成功。

4

安装并配置编辑器扩展

下载并安装扩展，然后在编辑器设置中开启 checkpoint 与 AI Tab 追踪：

下载 git-ai 扩展 
{
  "gitai.enableCheckpointLogging": true,
  "gitai.experiments.aiTabTracking": true
}

Trae 用户还需要在 Hooks 设置中手动开启“已配置的 Hooks”。扩展或 Hooks 未自动生效时，运行 git-ai install-hooks。

5

开始使用

无需为每个 Git 仓库单独初始化。直接使用支持的 AI 工具编辑代码，照常执行 git add、git commit 等 Git 命令；提交时会自动上传 AI 与人工代码归属统计。

查看当前归属统计：

git-ai status

需要更新时运行 git-ai update；遇到问题时运行 git-ai debug，并将输出发给管理员。

01

安装前准备

开始前完成下面三项检查。

1

检查 Git

在终端或 PowerShell 中运行：

git --version

看到类似 git version 2.x.x 的输出即可。如果提示找不到 Git，请先安装 Git。

2

检查服务器连接

macOS / Linux：

curl -fsS http://117.147.213.234:38080/health

Windows PowerShell：

Invoke-RestMethod http://117.147.213.234:38080/health

服务正常时会返回 status: ok。

3

注册账号

使用姓名、企业邮箱和密码注册，并选择所属组织与部门。注册后请保持浏览器登录状态。

打开注册页面 ↗
02

安装客户端

选择你的操作系统，安装程序会自动识别处理器架构。

⊞

Windows

打开 PowerShell，执行：

irm http://117.147.213.234:38080/worker/releases/latest/download/install.ps1 | iex

如 PowerShell 禁止运行脚本，先为当前窗口临时放行：

Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

安装后关闭并重新打开 PowerShell，再运行 git-ai --version。



macOS

打开“终端”，执行：

curl -fsSL \
  http://117.147.213.234:38080/worker/releases/latest/download/install.sh \
  | bash

关闭并重新打开终端，然后检查版本：

git-ai --version
◉

Linux

打开终端，执行：

curl -fsSL \
  http://117.147.213.234:38080/worker/releases/latest/download/install.sh \
  | bash

重新打开终端，或刷新当前窗口的 PATH：

export PATH="$HOME/.git-ai/bin:$PATH"
git-ai --version
03

登录与网页授权

安装完成后，将 CLI 与你的企业账号关联。

有图形界面的电脑

git-ai login --server http://117.147.213.234:38080

命令会打开浏览器。已有账号可直接登录并确认授权，也可以手动打开 企业登录页面 ↗。

没有图形界面的 Linux 服务器

git-ai login --server http://117.147.213.234:38080 --no-browser

复制终端显示的授权地址，在有浏览器的电脑上打开并完成授权。

检查登录状态

git-ai whoami

切换账号

git-ai logout
git-ai login --server http://117.147.213.234:38080
04

检查安装结果与更新

验证版本、账号、服务器地址和真实 Git 路径。

git-ai --version
git-ai whoami
git-ai config api_base_url
git-ai git-path
预期结果

服务器地址为 http://117.147.213.234:38080；git-ai git-path 显示真实 Git 路径，例如 macOS / Linux 上的 /usr/bin/git，或 Windows 上 Git for Windows 的 git.exe。

更新 Git AI

安装企业服务器上的最新版本：

git-ai update
git-ai --version

强制重新安装当前最新版：

git-ai update --force

也可以重新运行对应系统的安装命令。安装程序会覆盖旧版本，并保留现有登录和配置。

05

编辑器与 AI 工具集成

使用前请安装扩展 git-ai-0.1.22.vsix，并开启 checkpoint 与 AI Tab 追踪。

下载 git-ai 扩展 
{
  "gitai.enableCheckpointLogging": true,
  "gitai.experiments.aiTabTracking": true
}
Trae 用户注意

在 Hooks 设置中，手动开启“已配置的 Hooks”选项。

支持的编辑器与工具

VS CodeCursorTraeCodeBuddyQoderJClaude CodeCodexGitHub CopilotWindsurfGemini CLIOpenCodeAmpPiDroidFirebender

如果安装 CLI 后才安装新的编辑器或 AI 工具，或者 Hooks 自动配置失败，请运行 git-ai install-hooks。

06

日常使用

无需为每个 Git 仓库单独初始化，正常使用支持的 AI 工具编辑代码并执行 Git 命令即可。

数据如何上传

执行 git commit 时，AI 与人工代码归属统计数据会上传至统计后台。

查看归属统计

git-ai status

查看更多命令

git-ai help
07

常见问题

点击问题查看处理方法。

提示 git-ai: command not found

先关闭并重新打开终端。macOS / Linux 也可以临时执行：

export PATH="$HOME/.git-ai/bin:$PATH"
git-ai --version
提示无法找到标准 Git

运行下面两条命令。macOS 系统 Git 通常位于 /usr/bin/git；如果仍然报错，请把输出发给管理员。

git --version
command -v git
登录页面没有自动打开

重新运行登录命令；如果仍未打开，使用无浏览器模式并手动复制授权地址。

git-ai login --server http://117.147.213.234:38080 --no-browser
无法连接企业服务器

macOS / Linux：

curl -v http://117.147.213.234:38080/health

Windows PowerShell：

Test-NetConnection 117.147.213.234 -Port 38080

如果连接失败，请确认当前网络允许访问 117.147.213.234:38080，必要时联系管理员。

macOS 提示开发者或安全限制

请确认安装命令来自本页企业服务器地址。安装器会尝试移除下载文件的隔离属性；如果仍被系统阻止，请将完整提示截图发给管理员。

Windows 安装后仍找不到 git-ai

关闭所有 PowerShell 或命令提示符窗口，重新打开后运行 git-ai --version。如果仍然找不到，请确认用户 PATH 包含：

%USERPROFILE%\.git-ai\bin
Hooks 自动配置失败或新安装了编辑器

重新执行 Hooks 安装：

git-ai install-hooks
08

获取诊断信息

遇到问题时运行诊断命令，并将结果交给管理员。

git-ai debug

请同时提供操作系统版本、git-ai --version 输出和报错截图，以便更快定位问题。
