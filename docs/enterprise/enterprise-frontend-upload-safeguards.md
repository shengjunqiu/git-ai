# Enterprise Dashboard 管理上传保护

日期：2026-07-20

## 目标

阶段 5.5 在文件进入网络请求前提供可理解的校验和大小反馈，并防止重复提交与误离开
页面。浏览器校验只用于尽早反馈，服务端仍是最终安全边界。

## 客户端行为

### CLI 发布包

- 必须且只能选择六个平台文件。
- 文件名必须与发布协议完全一致：
  - `git-ai-linux-x64`
  - `git-ai-linux-arm64`
  - `git-ai-windows-x64.exe`
  - `git-ai-windows-arm64.exe`
  - `git-ai-macos-x64`
  - `git-ai-macos-arm64`
- 拒绝缺失、多余、重复、空文件、单文件超过 100 MiB 或总计超过 500 MiB。
- 选择区域显示每个文件大小、选择数量、总大小和可上传状态。
- 版本号在提交前按服务端字符规则检查。

### 文件中心

- 每次只能选择一个文件。
- 文件名不能为空、不能是 `.` 或 `..`、不能包含路径分隔符或 NUL，UTF-8 长度不能
  超过 240 字节。
- 拒绝空文件和超过 500 MiB 的文件。
- 选择区域显示文件名、大小、扩展名和上限。
- 文件中心用于发布文档、配置、压缩包及其他企业制品，因此不新增扩展名白名单；
  无扩展名文件也会明确显示为“无扩展名”。
- 文件标识和版本号在提交前按服务端规则检查。

### 提交生命周期

- CLI 发布与普通文件上传分别使用 single-flight 状态，同一操作进行中时后续提交直接
  被拒绝。
- 上传按钮在操作期间禁用并显示“正在上传...”，结束后恢复原标签。
- 任一上传进行中时，关闭或刷新页面会触发浏览器原生离页确认。
- 状态区域显示上传文件数量、单文件大小或总大小。
- 上传请求超时从 2 分钟调整为 15 分钟，避免合法的大文件在较慢企业网络中被客户端
  过早中止。

## 服务端最终边界

- CLI 二进制单文件上限：100 MiB。
- CLI 六文件总计上限：500 MiB。
- 普通文件单文件上限：500 MiB。
- Multipart 请求体上限继续为 512 MiB，为 500 MiB 文件保留表单字段和边界开销。
- 发布接口在读取文件内容前拒绝非协议文件名和重复文件名，读取后再次检查空文件、
  单文件大小和累计大小。
- 普通文件接口拒绝多个 `file` 字段，并在存储前重复执行文件名和大小校验。

## 进度与取消评估

当前请求层使用 `fetch + multipart`，浏览器没有稳定的上传进度事件。客户端中止请求也
不能保证撤销服务端已经开始的对象存储或数据库写入，因此本阶段不提供可能误导用户的
假进度条或“取消即回滚”按钮。

如果实际部署中 500 MiB 上传成为常态，后续应使用分片/可恢复上传协议，并由服务端
提供 upload ID、已接收字节数、显式取消与清理状态；届时再增加可靠进度和取消操作。


结果：

- 前端行为测试 28/28 通过。
- Enterprise Server 测试 174/174 通过。
- 新增覆盖包括发布文件协议、重复/空/超限文件、累计大小、普通文件名与扩展名反馈、
  single-flight 重复提交保护、按钮恢复和离页确认。

## 兼容性

- 已发布文件和现有数据库记录不受影响。
- 现有 512 MiB HTTP 请求体边界不变；新增 500 MiB 业务上限避免 multipart 开销在
  边界处产生不明确失败。
- 普通文件类型保持开放，不会因为新增前端检查而禁止原有制品格式。
